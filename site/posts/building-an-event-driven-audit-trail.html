Building an Event-Driven Audit Trail with S3 and Lambda

JF
Jay French
Sep 28, 2025  ·  4 min read

This week, I built a simple yet powerful workflow where an S3 upload automatically triggers a Lambda function. The function logs file details and writes an “audit record” back into the same bucket. This pattern is at the core of modern cloud solutions: lightweight, scalable, and serverless.

💡
TL;DR: I built an event-driven audit system where S3 uploads trigger a Lambda function that logs file metadata and writes a JSON audit record back to the bucket. This serverless pattern is cheap, scalable, and useful for compliance, monitoring, and automation, and it can easily extend to workflows like virus scanning, validation, or notifications.

Why This Matters

Event-driven systems let you react to changes in real time without constant polling. Instead of writing a script to scan for uploads, we let S3 events drive the workflow. This:

  • Keeps costs near zero
  • Improves responsiveness
  • Simplifies operations

And, by writing an audit record back into S3, we’re building a durable log of every upload, something compliance, debugging, and monitoring teams love.

Architecture Overview

  • Amazon S3 Bucket — Stores uploaded files. Configured to fire ObjectCreated events.
  • AWS Lambda (Python 3.12) — Triggered by uploads, it records file metadata and writes a JSON audit file under /audit/.
  • IAM Role — Provides Lambda with least-privilege access to read objects, write audit records, and push logs.
  • CloudWatch Logs — Captures function execution details.

Workflow

  • File uploaded → S3 sends event → Lambda executes.
  • Lambda extracts metadata and writes audit/.json.
